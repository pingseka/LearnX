## 遇到的问题和解决

1. 问题：服务器启动时 uploads 目录不存在导致文件上传失败
   解决：在 app.ts 启动时添加自动创建 uploads 目录的逻辑
2. 问题：跨域请求被浏览器阻止
   解决：配置 CORS 中间件，允许前端域名访问
3. 问题：API 请求频率过高导致服务器压力过大
   解决：实现请求限流中间件，限制 100 次/15 分钟
4. 问题：错误响应格式不统一，前端处理困难
   解决：实现统一的错误处理中间件，确保所有接口返回一致的响应格式
5. 问题：素材更新和删除时权限验证缺失
   解决：在控制器中添加素材作者权限验证，确保用户只能修改和删除自己的素材
